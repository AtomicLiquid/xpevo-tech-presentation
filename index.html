The power of event-driven architecture

Plain English: One source pushes live updates instantly — no polling.

  • 📡 Instant updates to overlay, admin panel and services
  • 🔁 Decouples game servers (producers) from consumers
  • ⚡ Handles bursty event loads efficiently

Technical details

  • Transport: NATS JetStream — lightweight, low-latency pub/sub with persistence
  • Events: semantic topics like race.finish, player.join, server.status
  • Design: Subscribers subscribe only to needed streams for performance & isolation

Flow: player finishes → EvoSC publishes race.finish → Overlay updates + Results are published to Toornament.

🚧 Challenges We Faced

Real-time Performance

Challenge: Processing hundreds of events per second

Solution: Efficient event handling and caching

Tournament software

Challenge: Finding a platform that supports Trackmania's Tournament 1v1v1v1 format

Temporary solution: Utilize Toornament's platform

Long-term solution: Develop a custom solution tailored to our needs

Testing

Challenge: Simulating real tournament conditions

Solution: Local test environment with mock data and small-scale trials

Reality: Nothing beats the chaos of a live event! 🔥
